# Backend — AGENTS.md

Go REST API for RatioDash. Serves JSON over HTTP, scrapes torrent-site stats on a per-site cron schedule, and persists everything in SQLite.

---

## Directory layout

```
backend/
├── cmd/api/main.go          # FX application entry-point — wires every module
├── pkg/
│   ├── config/              # Env-based config (SERVER_ADDR, DATABASE_URL)
│   └── database/            # GORM connection + Goose migration runner
├── migrations/              # *.sql files embedded via //go:embed *.sql
└── internal/
    ├── domain/              # Entities + repository/service interfaces
    ├── repository/          # GORM implementations  ← see repository/AGENTS.md
    ├── service/             # Business logic
    ├── scraper/             # Registry + per-site HTTP adapters
    ├── scheduler/           # robfig/cron wrapper (implements domain.Refresher)
    ├── handler/             # Huma v2 HTTP handlers  ← see handler/AGENTS.md
    └── server/              # chi router + HTTP server lifecycle hook
```

---

## FX wiring order

```
config → database → repository → scraper → service → scheduler → handler → server
```

Every package under `internal/` and `pkg/` exports `var Module = fx.Options(…)`. `cmd/api/main.go` composes them in the order above. To add a new concern, create a package with its own `Module` and append it to `main.go`.

---

## Layer responsibilities

| Layer | Package | Responsibility |
|---|---|---|
| Config | `pkg/config` | Read env vars; expose a typed `Config` struct |
| Database | `pkg/database` | Open GORM connection; run Goose migrations at startup |
| Domain | `internal/domain` | Define entities (`Site`, `SiteStats`) and interfaces (`SiteRepository`, `StatsRepository`, `SiteService`, `StatsService`, `RefreshService`, `ScraperRegistry`, `SiteScraper`) |
| Repository | `internal/repository` | GORM implementations of domain repository interfaces; no business logic |
| Service | `internal/service` | Orchestrate repositories; enforce invariants; default field values |
| Scraper | `internal/scraper` | Per-site HTTP adapters; registry; shared HTTP helpers |
| Scheduler | `internal/scheduler` | robfig/cron wrapper; schedules/unschedules `RefreshService.RefreshSite` per site |
| Handler | `internal/handler` | Huma v2 HTTP handlers; delegate to domain services; no business logic |
| Server | `internal/server` | Mount chi router; configure CORS; start/stop HTTP server via FX lifecycle |

## Domain types

### `Site`
Represents a tracked torrent site.

| Field | Type | Notes |
|---|---|---|
| `ID` | `uint` | Auto-increment primary key |
| `Name` | `string` | Display name |
| `URL` | `string` | Tracker homepage URL |
| `ScraperKey` | `string` | Matches `SiteScraper.Key()` |
| `Credentials` | `string` | JSON blob — stored encrypted-at-rest via SQLite; excluded from API responses (`json:"-"`) |
| `CronExpr` | `string` | robfig/cron expression (default `@hourly`) |
| `Active` | `bool` | Whether the scheduler runs this site |
| `CreatedAt`, `UpdatedAt` | `time.Time` | GORM auto-managed |

### `SiteStats`
A point-in-time snapshot created on every successful scrape.

| Field | Type | Notes |
|---|---|---|
| `ID` | `uint` | Auto-increment primary key |
| `SiteID` | `uint` | FK → `sites.id` (CASCADE DELETE) |
| `Uploaded` | `int64` | Raw bytes |
| `Downloaded` | `int64` | Raw bytes |
| `Ratio` | `float64` | Computed by the scraper (or `uploaded/downloaded`) |
| `FetchedAt` | `time.Time` | UTC timestamp of the scrape |

## API routes

All routes are under `/api/v1/`. The full OpenAPI 3.1 spec is auto-generated by Huma and served at `/docs`.

| Method | Path | Description |
|---|---|---|
| `GET` | `/sites` | List all sites |
| `POST` | `/sites` | Create site — runs initial scrape synchronously; rolls back on failure |
| `PATCH` | `/sites/{id}` | Partial update |
| `DELETE` | `/sites/{id}` | Delete site + cascade stats |
| `POST` | `/sites/{id}/refresh` | Trigger immediate scrape |
| `GET` | `/dashboard` | Latest stats snapshot for every site (null if none yet) |
| `GET` | `/sites/{site_id}/stats` | History for a site (default limit 50, `?limit=N`) |
| `GET` | `/sites/{site_id}/stats/latest` | Most recent snapshot |
| `DELETE` | `/sites/{site_id}/stats/{stat_id}` | Delete a single history entry |
| `GET` | `/scrapers` | List scraper keys + credential field definitions |
| `GET` | `/health` | Health check |
| `GET` | `/docs` | Swagger UI |

## Scraper layer

### Interface

```go
type SiteScraper interface {
    Key()              string
    CredentialFields() []CredentialField   // drives the frontend form
    Fetch(ctx context.Context, site Site) (*SiteStats, error)
}
```

`SiteID` and `FetchedAt` are set by the caller (`RefreshService`) after `Fetch` returns; the scraper only fills `Uploaded`, `Downloaded`, and `Ratio`.

### Registered scrapers

| Key | File | Auth method |
|---|---|---|
| `unit3d` | `scraper/unit3d.go` | Bearer API token → `GET /api/user` |
| `c411` | `scraper/c411.go` | Cookie-jar login + CSRF → `GET /api/auth/me` |
| `torr9` | `scraper/torr9.go` | JWT login → `GET /api/v1/users/me` |

### `BaseScraper` helpers (`scraper/base.go`)

Embed `BaseScraper` to get:
- `Get(ctx, site)` — authenticated GET to `site.URL`; sets Cookie, Bearer, X-API-Key, and custom headers from `Credentials`
- `DoRequest(ctx, method, url, creds)` — lower-level; returns human-readable errors for 401/403/404/5xx
- `ParseJSON(body, dst)` — detects HTML-instead-of-JSON and returns a clear error
- `ParseXML(body, dst)` — XML response parsing
- `ParseHTML(body)`, `WalkHTML(node, fn)`, `HTMLAttr(node, key)` — HTML tree helpers

`Credentials` struct (parsed from `Site.Credentials` JSON):

```go
type Credentials struct {
    Cookie   string
    Username string
    Password string
    APIKey   string
    Token    string
    Headers  map[string]string
}
```

### Adding a new scraper

1. Create `internal/scraper/{key}.go`. Implement `Key()`, `CredentialFields()`, and `Fetch()`.  
   - For simple JSON APIs, embed `BaseScraper` and use `Get` + `ParseJSON`.  
   - For login flows (CSRF, cookies), create a fresh `http.Client` with `cookiejar.New(nil)` per `Fetch()` call.
2. Register in `internal/scraper/module.go`:

```go
fx.Provide(
    fx.Annotate(
        NewMyScraper,
        fx.As(new(domain.SiteScraper)),
        fx.ResultTags(`group:"scrapers"`),
    ),
),
```

3. FX automatically injects all scrapers tagged `group:"scrapers"` into the registry. No other wiring needed.

## Scheduler

`internal/scheduler` wraps `robfig/cron/v3`. It implements `domain.Refresher`:

```go
type Refresher interface {
    Schedule(site domain.Site)
    Unschedule(siteID uint)
}
```

- `Schedule` parses `site.CronExpr` and registers a job that calls `RefreshService.RefreshSite`.
- `Unschedule` removes the job for that `siteID`.
- The scheduler starts via an FX `OnStart` hook and stops via `OnStop`.
- Only sites with `Active = true` are scheduled.

## Adding a new entity

1. Define the struct + repository/service interfaces in `internal/domain/{entity}.go`.
2. Implement the repository in `internal/repository/{entity}_repository.go` — follow [repository/AGENTS.md](internal/repository/AGENTS.md).
3. Implement the service in `internal/service/{entity}_service.go`.
4. Write handlers in `internal/handler/{entity}_handler.go` — follow [handler/AGENTS.md](internal/handler/AGENTS.md).
5. Add a migration in `migrations/000N_create_{entities}.sql`.
6. Wire constructors into the relevant `module.go` files.

---

## Migrations

SQL files live in `migrations/` and are embedded at compile time via `//go:embed *.sql` in `migrations/embed.go`. Goose runs them automatically at startup (up-only). Migration filenames follow the pattern `000N_description.sql` with `-- +goose Up` / `-- +goose Down` markers.
