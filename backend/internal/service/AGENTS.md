# Service layer — naming conventions

## Files & types

| Element | Convention | Example |
|---|---|---|
| File | `{entity}_service.go` | `tracker_service.go` |
| Struct (unexported) | `{entity}Service` | `trackerService` |
| Constructor | `New{Entity}Service(deps…) domain.{Entity}Service` | `NewTrackerService` |
| Domain interface | `{Entity}Service` in `internal/domain/` | `TrackerService`, `StatsService` |
| FX registration | `fx.Provide(New{Entity}Service)` in `module.go` | |

## Method names

| Pattern | Signature |
|---|---|
| All rows | `GetAll() ([]T, error)` |
| By primary key | `GetByID(id uint) (*T, error)` |
| Filtered list | `GetBy{Field}(val) ([]T, error)` |
| Insert | `Create(input domain.Create{Entity}Input) (*T, error)` |
| Partial update | `Update(id uint, input domain.Update{Entity}Input) (*T, error)` |
| Remove | `Delete(id uint) error` |

## Rules

- **Enforce invariants**: apply defaults (e.g. `CronExpr = "@hourly"`, `Credentials = "{}"`) and validate inputs before passing them to repositories.
- **Not-found errors**: when a repository returns `nil, nil`, convert it to a descriptive error (e.g. `fmt.Errorf("tracker %d not found", id)`). Handlers translate named sentinel errors to 404s.
- **Credential redaction**: always populate `PublicCredentials` via `domain.RedactCredentials` before returning entities to callers. Never expose raw credential JSON.
- **No direct DB access**: services depend only on domain repository interfaces, never on `*gorm.DB`.
- **Error wrapping**: add context — `fmt.Errorf("creating tracker: %w", err)`.
- **No HTTP concerns**: services must not import `net/http` or reference HTTP status codes.

## Services

| File | Interface | Responsibility |
|---|---|---|
| `tracker_service.go` | `domain.TrackerService` | CRUD for trackers; default field values; credential redaction |
| `stats_service.go` | `domain.StatsService` | Stats history, latest snapshot, dashboard aggregation |
| `refresh_service.go` | `domain.RefreshService` | Orchestrate scraper fetch → persist stats → update scrape status |
| `auth_service.go` | `domain.AuthService` | Setup credentials, bcrypt login, JWT issuance |

## Testing conventions

### File & package

| Element | Convention | Example |
|---|---|---|
| File | `{entity}_service_test.go` | `tracker_service_test.go` |
| Package | `service_test` (black-box) | — |

Use the external `service_test` package so tests exercise only the public `domain.{Entity}Service` interface, not internal struct fields.

### Mocks

Mocks are generated by [mockery v2](https://vektra.github.io/mockery/) from the domain interfaces. Generated files live in `internal/mocks/` and must never be edited by hand.

#### Configuration

`.mockery.yaml` at the repo root:

```yaml
with-expecter: true
dir: internal/mocks
outpkg: mocks
packages:
  github.com/jose/ratiodash/internal/domain:
    interfaces:
      TrackerRepository:
      StatsRepository:
      AuthRepository:
      TrackerService:
      StatsService:
      ScraperRegistry:
      TrackerScraper:
```

#### Regenerating

```sh
mockery
```

Re-run whenever a domain interface changes. Commit the generated files.

### Setup

Construct the service with generated mocks via their `New*` constructors, configure expectations with the type-safe `.EXPECT()` API, and assert with `mock.AssertExpectations(t)` at the end.

```go
func TestTrackerService_GetAll(t *testing.T) {
    repo := mocks.NewTrackerRepository(t)
    repo.EXPECT().FindAll().Return(
        []domain.Tracker{{Name: "MyTracker", Credentials: `{"api_key":"secret"}`}},
        nil,
    )

    svc := service.NewTrackerService(repo)

    trackers, err := svc.GetAll()

    require.NoError(t, err)
    assert.Len(t, trackers, 1)
}
```

Passing `t` to `New*` registers `AssertExpectations` automatically — no manual call needed.

### Structure

- **One `Test*` function per service method** — keep each function focused on a single method (e.g. `TestTrackerService_Create`, `TestTrackerService_GetByID`).
- **Sub-tests for cases** — use `t.Run("case name", ...)` for happy path, not-found, validation failure, repository error, etc.
- **Table-driven only when inputs vary** — prefer explicit sub-tests over tables when side-effects accumulate across rows.
- **Create a fresh mock per sub-test** — never reuse a mock across `t.Run` boundaries.

### Assertions

Use `github.com/stretchr/testify/assert` for non-fatal checks and `require` for fatal ones (first failure that makes subsequent assertions meaningless).

```go
require.NoError(t, err)
assert.Equal(t, "@hourly", tracker.CronExpr)   // default applied
assert.Empty(t, tracker.Credentials)            // raw creds not exposed
assert.NotEmpty(t, tracker.PublicCredentials)   // redacted form present
assert.ErrorContains(t, err, "not found")
```

### What to test

| Scenario | Required |
|---|---|
| Happy path | yes |
| Default field values applied (e.g. `CronExpr`, `Credentials`) | yes — in `Create` |
| Raw credentials never returned; `PublicCredentials` populated | yes — for all read methods |
| Not-found converted to a descriptive error (not `nil, nil`) | yes — for `GetByID` and similar |
| Invalid/missing inputs return an error | yes |
| Repository error propagated as an error | yes — every method |
| HTTP concerns (status codes, request parsing) | **no** — belongs in handler tests |

### Fixture helpers

Define unexported helpers inside the test file to build minimal valid inputs:

```go
func newCreateInput(name string) domain.CreateTrackerInput {
    return domain.CreateTrackerInput{
        Name:       name,
        ScraperKey: "generic",
    }
}
```

### RefreshService

`RefreshService` depends on `domain.TrackerService`, `domain.StatsRepository`, `domain.TrackerRepository`, and `domain.ScraperRegistry` — mock all four. Focus tests on the orchestration branches:

- Scraper not found → error returned, `UpdateScrapeStatus` called with the error message.
- `Fetch` fails → error returned, `UpdateScrapeStatus` called with the error message.
- `StatsRepository.Create` fails → error returned, `UpdateScrapeStatus` called.
- Success → `StatsRepository.Create` called with correct `TrackerID`, `FetchedAt` set, ratio rounded to 2 dp, `UpdateScrapeStatus` called with `""`.

```go
func TestRefreshService_RefreshTracker_FetchError(t *testing.T) {
    trackerSvc  := mocks.NewTrackerService(t)
    statsRepo   := mocks.NewStatsRepository(t)
    trackerRepo := mocks.NewTrackerRepository(t)
    registry    := mocks.NewScraperRegistry(t)
    scraper     := mocks.NewTrackerScraper(t)

    tracker := &domain.Tracker{ID: 1, ScraperKey: "unit3d"}
    fetchErr := errors.New("network timeout")

    trackerSvc.EXPECT().GetByID(uint(1)).Return(tracker, nil)
    registry.EXPECT().Get("unit3d").Return(scraper, true)
    scraper.EXPECT().Fetch(mock.Anything, *tracker).Return(nil, fetchErr)
    trackerRepo.EXPECT().UpdateScrapeStatus(uint(1), fetchErr.Error()).Return(nil)

    svc := service.NewRefreshService(trackerSvc, statsRepo, trackerRepo, registry)
    err := svc.RefreshTracker(context.Background(), 1)

    assert.ErrorContains(t, err, "network timeout")
}
```

### Coverage requirement

Tests must achieve **100% statement coverage** of the service implementation file. Run with:

```sh
go test ./internal/service/... -coverprofile=cover.out && go tool cover -func=cover.out
```

All uncovered lines are a test gap that must be addressed before merging.

## RefreshService specifics

`RefreshService.RefreshTracker` is the single entry-point for on-demand and scheduled scrapes:

1. Resolve the tracker via `TrackerService.GetByID`.
2. Look up the scraper in `ScraperRegistry`.
3. Call `SiteScraper.Fetch` — on error, record the failure via `TrackerRepository.UpdateScrapeStatus` before returning.
4. Stamp `TrackerID`, `FetchedAt` (UTC), and round `Ratio` to 2 decimal places.
5. Persist via `StatsRepository.Create` — on error, record the failure before returning.
6. Clear the scrape status on success via `TrackerRepository.UpdateScrapeStatus("")`.
